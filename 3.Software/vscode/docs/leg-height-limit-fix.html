腿高极限位优化

最高处抖动 / 最低处电机持续使能 — 详细修改说明 · 固件 ver1.7.1+

1. 问题现象

工况现象用户观察
机身调节到最高（方向键 ↑ 多次） 轮子高频振荡，机身明显抖动 LQI 增益过大 + 俯仰反馈过激
机身调节到最低（方向键 ↓ 多次） 静止时电机持续输出，空载电流偏高 LQI 持续追不可消除的 IMU 倾角
核心判断（与用户分析一致）：在腿高机械极限处，关节已无法继续伸缩来调平机身，IMU 始终存在结构性俯仰偏置。此时 LQI 仍把该偏置当作"需要纠正的误差"，驱动轮子持续输出扭矩，导致抖动或电机常使能。

2. 根因分析

2.1 舵机到达机械限位，但控制目标仍在变化

修改前，leg_height_base 可在约 −10（最高）～ 52（最低）范围内无限制调节。当目标超出舵机行程时：

left_position  = constrain(..., SERVO_LEFT_MIN, SERVO_LEFT_MAX)
right_position = constrain(..., SERVO_RIGHT_MAX, SERVO_RIGHT_MIN)

舵机位置被钳位不再移动，但 IMU 读到的俯仰角仍偏离平衡零点，LQI 无法区分"可纠正误差"与"机械固有倾角"。

2.2 LQI 增益在标定范围外被外推

lqi.cpp 中定义了腿高标定范围：

car.leg_min_height = 0.02f;  // 2 cm
car.leg_max_height = 0.06f;  // 6 cm

修改前，update_feedback_gain() 直接用实测腿高 h 代入三次多项式求增益，未做钳位。超出 2～6 cm 时多项式外推，增益偏大，最高处稳定性变差。

2.3 俯仰状态直接进入全增益反馈

LQI 状态向量中：

索引变量极限位时的问题
x[0]pitch_angle反映 IMU 固有倾角，不可被轮子消除
x[1]pitch_rate轮子追角度时产生振荡，表现为抖动
x[2]~x[5]速度/偏航/积分静止时影响较小，保留全增益
修改前控制链路： 方向键调节 leg_height_base │ ▼ 舵机目标 ──► 撞机械限位 ──► 实际腿高不再变化 │ ▼ IMU 俯仰偏置（不可消除）──► x[0], x[1] 进入 LQI │ ▼ 增益 K(h) 外推偏大 ──► 轮子大力矩输出 ──► 抖动 / 持续使能

3. 优化策略：三层死区

采用分层防护，从"不让用户进入危险区"到"进入后降低控制激进度"：

层级名称作用修改位置
第一层 调节死区 限制 leg_height_base 在安全范围，用户无法调到机械极限 sts3032.h、leg_loop()
第二层 增益钳位 LQI 增益仅在标定腿高 [0.02, 0.06] m 内插值，禁止外推 update_feedback_gain()
第三层 俯仰衰减 接近极限或舵机撞限位时，降低 x[0]、x[1] 反馈权重 get_height_limit_blend()、lqi_loop()
三层互为补充：第一层减少进入极限的概率；第二层防止增益异常；第三层在已处于极限时主动降低俯仰环激进度，直接针对抖动和持续使能现象。

4. 第一层：调节死区

文件：lib/devices/sts3032.h、lib/controller/controller.cpp

4.1 新增宏定义

// 安全调节死区：距机械极限留出余量（可按实机微调）
#define LEG_HEIGHT_BASE_MIN  5     // 机身最高侧下限（leg_height_base 不可再小）
#define LEG_HEIGHT_BASE_MAX  40    // 机身最低侧上限（leg_height_base 不可再大）

对比原实测范围 −10（最高）～ 52（最低），两侧各留出约 15 个单位的死区余量。默认值 20 仍在安全区内。

4.2 leg_loop 中强制钳位

if((buttons & BTN_UP)   && !(buttons & ~BTN_UP))   { leg_height_base -= 0.025f; }
if((buttons & BTN_DOWN) && !(buttons & ~BTN_DOWN)) { leg_height_base += 0.025f; }
leg_height_base = constrain(leg_height_base,
    (float)LEG_HEIGHT_BASE_MIN, (float)LEG_HEIGHT_BASE_MAX);

方向键每次步进 0.025，到达边界后不再响应，舵机不会继续逼近机械限位。

5. 第二层：增益钳位

文件：lib/controller/controller_base.cpp — update_feedback_gain()

5.1 修改内容

void controller_base::update_feedback_gain(float height)
{
    // LQI 增益仅在标定腿高范围内插值，避免极限位外推导致增益过大/抖动
    const float h = constrain(
        height,
        ctrl->lqi_param.car.leg_min_height,   // 0.02 m
        ctrl->lqi_param.car.leg_max_height    // 0.06 m
    );

    if(fabsf(h - last_height) < 1.0e-4f) { return; }
    last_height = h;
    // ... 三次多项式求 K[i][j] ...
}

5.2 效果

  • 实测腿高 < 2 cm 时，增益按 2 cm 处计算（不再外推更小值）
  • 实测腿高 > 6 cm 时，增益按 6 cm 处计算（不再外推更大值）
  • 最高处增益不再异常放大，从根本上抑制振荡趋势

    ...
}

用于检测"舵机已撞限位但 leg_height_base 尚未到调节死区"的边缘情况。

6.2 新增函数：get_height_limit_blend()

根据实测腿高 h（米）和舵机限位状态，返回俯仰衰减系数 blend ∈ [0.35, 1.0]。详见第 7 节。

6.3 lqi_loop 中应用俯仰衰减

const float leg_h = base_components.leg_servo_count_to_height();
base_components.update_feedback_gain(leg_h);
// ... 组装 x[6] ...

// 腿高极限位 IMU 存在不可消除的机械倾角，衰减俯仰反馈
if(!balance_recover_active && !jump_active)
{
    const float pitch_blend = base_components.get_height_limit_blend(leg_h);
    x[0] *= pitch_blend;   // pitch_angle
    x[1] *= pitch_blend;   // pitch_rate
}

6.4 设计取舍

决策原因
只衰减 x[0]、x[1]，不碰 x[2]~x[5] 速度/偏航通道与腿高极限无关，保留正常行驶能力
最低保留 35% 俯仰增益，不完全置零 完全关闭俯仰环会导致极限位失稳倒下
balance_recover / jump 模式不启用衰减 恢复站立和跳跃需要全增益俯仰控制

7. blend 算法详解

7.1 分段逻辑

实测腿高 h（米） blend 值 ───────────────────────────────────────── h ≤ h_min (0.02) 0.35（最低） h_min ~ h_min+4mm 0.35 → 1.0 线性上升 中间安全区 1.0（全增益） h_max-4mm ~ h_max (0.06) 1.0 → 0.35 线性下降 h ≥ h_max 0.35（最低） 若舵机撞机械限位：blend = min(blend, 0.45)

7.2 关键参数

参数默认值含义
h_min0.02 mLQI 标定最低腿高（机身最高侧）
h_max0.06 mLQI 标定最高腿高（机身最低侧）
margin0.004 m (4 mm)过渡区宽度，越大越早开始衰减
min_blend0.35极限位最低俯仰增益比例
限位判定 margin10 计数舵机位置距 MIN/MAX 的判定阈值
限位附加衰减0.45撞限位时 blend 上限

7.3 衰减示意（最低处，h ≈ 0.06 m）

假设 IMU 读到俯仰角 0.08 rad，原 LQI 俯仰输出较大。施加 blend = 0.35 后：

x[0]_effective = 0.08 × 0.35 = 0.028 rad  （等效误差降低 65%）
→ 轮子输出扭矩大幅下降 → 电机不再持续大电流

9. 实机调参指南

9.1 调节死区范围（sts3032.h）

现象调整方向示例
最高处仍抖动 增大 LEG_HEIGHT_BASE_MIN（不让机身升太高） 5 → 8 → 10
最低处电机仍持续使能 减小 LEG_HEIGHT_BASE_MAX（不让机身降太低） 40 → 35 → 32
安全范围太窄，不够用 适当放宽，同时调大 margin 和降低 min_blend MIN=3, MAX=42

9.2 俯仰衰减参数（controller_base.cpp → get_height_limit_blend）

参数仍抖时要倒了时
margin增大到 0.006减小到 0.003
min_blend降低到 0.25提高到 0.45
限位附加 0.45降低到 0.35提高到 0.55

9.3 调参顺序建议

  1. 先调 LEG_HEIGHT_BASE_MIN/MAX，找到不抖、不持续使能的安全高度范围
  2. 若安全范围满意但边缘仍有轻微问题，再调 margin 和 min_blend
  3. 每次只改一个参数，改完烧录实机验证

10. 验证方法

10.1 功能验证

步骤操作预期结果
1 平衡模式下连续按 ↑ 抬高机身 到达最高安全位后不再升高；抖动明显减轻或消失
2 平衡模式下连续按 ↓ 降低机身 到达最低安全位后不再降低；静止时电机电流下降
3 在安全高度范围内行驶 前进/后退/转向正常，与修改前无明显差异
4 按 RB 重新站立（balance_recover） 恢复流程正常，不受俯仰衰减影响
5 执行原地跳（RS） 跳跃流程正常，不受俯仰衰减影响
6 按 LS 复位 腿高基准回到 20，Roll 补偿清零

10.2 调试观测（可选）

固件已将状态量写入调试变量，可通过串口观测：

  • x_debug[0]、x_debug[1] — 衰减后的俯仰状态
  • output_debug[0]、output_debug[1] — 左右轮输出

在极限位时，x_debug[0/1] 应明显小于原始 IMU 俯仰值（约为 35%～45%）。

11. 后续可选优化

若三层死区仍无法完全满足需求，可考虑更进一步的方案：

方案 A：高度相关俯仰偏置补偿

在极限位实测 IMU 俯仰零点偏移，建立 pitch_bias(h) 查表或拟合曲线，反馈时使用 pitch - pitch_bias(h) 代替原始俯仰角。可更精准消除结构性倾角，但需要逐台标定。

方案 B：极限位自动回退

检测到舵机撞限位且 |pitch| 持续偏大时，自动将 leg_height_base 向中间值微调 1～2 步，脱离极限区。用户体验更好，但可能引起高度漂移感。

方案 C：重新标定 LQI 增益

将 LQI 增益多项式扩展到实际使用的全腿高范围（含极限区），从控制理论层面覆盖更宽工况。工作量大，需离线建模与仿真。
