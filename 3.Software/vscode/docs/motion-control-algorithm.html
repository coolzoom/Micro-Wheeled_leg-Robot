1. 系统概述

本机器人是一台轮腿式自平衡小车：两轮差速驱动负责前进、后退与转向；两条 STS3032 舵机腿负责机身高度与侧倾（Roll）补偿。核心平衡控制采用 LQI（Linear Quadratic Integral）——在 LQR 状态反馈基础上，对线速度与偏航角速度引入积分项，消除稳态跟踪误差。

控制对象（6 维状态向量）
索引符号物理含义来源

x₁θ̇俯仰角速度 pitch_rate（rad/s）MPU6050 陀螺仪

左右轮扭矩指令，经 SimpleFOC 以电压模式输出：

uL = Σj=0..5 KL,j(h) · xj   |   uR = Σj=0..5 KR,j(h) · xj

其中增益矩阵 K 随腿高 h 实时调度（见第 5 节）。

4.3 每周期执行顺序

在 fsm::loop() 中，先执行当前模式的专属逻辑（如 leg_loop、jump_loop），再统一调用 set_cam_angle() 与 lqi_loop()。即：无论何种模式，LQI 循环始终运行，但通过 enable_balance、enable_steering、enable_motor 标志位控制是否真正输出力矩。

5. LQI 平衡算法

源码：lib/controller/algorithm/lqi.cpp、controller.cpp::lqi_loop()

5.1 算法原理

LQI 在经典倒立摆 LQR 基础上扩展：对速度通道引入积分状态，使机器人在有负载扰动或地面摩擦时仍能准确跟踪摇杆给出的线速度与转向指令。

u = K(h) · [θ, θ̇, v−vref, ω−ωref, ∫ev, ∫eω]T

5.2 增益调度（腿高自适应）

腿高变化会改变整机质心与等效摆长，因此反馈增益随当前腿高 h 用三次多项式实时计算：

K[i][j] = c₃·h³ + c₂·h² + c₁·h + c₀

12 组多项式系数（每组 4 个）离线标定，存储于 lqi_gain_poly[12][4]。左右轮各用 6 个增益，对应 6 维状态。

腿高 h 由两侧舵机编码器计数经标定多项式反算（leg_servo_count_to_height()）：

dCount = (|pos_L − 2048| + |pos_R − 2048|) / 2
h = ((4.63e-12·dCount − 9.39e-08)·dCount + 1.54e-04)·dCount + 4.20e-02

5.3 lqi_loop 执行流程

  1. 更新增益矩阵 update_feedback_gain(h)
  2. 读取摇杆，死区处理后映射为目标线速度 / 转向角速度
  3. 生成平滑参考 ref.linear_vel、ref.yaw_rate 及积分状态
  4. 组装状态向量 x[6]，按模式屏蔽部分通道（跳跃/恢复时）
  5. 矩阵乘法得 output[0/1]，乘以 output_blend 后送电机

5.4 摇杆输入映射

摇杆轴死区映射备注
axes[3]（右摇杆纵轴）5%±0.6 m/s 线速度后退 ×0.8
axes[0]（左摇杆横轴）5%±4.0 rad/s 偏航率参考取负号

死区函数 apply_axis_deadband：在死区内输出 0；超出后线性重映射到 [−1, 1]，保证边界处连续。

6. 运动参考生成

源码：lib/controller/controller_base.cpp

6.1 线速度参考 update_linear_reference

采用两级平滑：摇杆目标先经一阶低通（τ=24 ms），再经斜率限制生成最终参考 vref。

参数值含义
max_ref_accel1.60 m/s²加速斜率上限
max_ref_decel3.10 m/s²减速斜率上限
max_release_decel7.20 m/s²松杆后快速归零斜率
release_duration0.45 s松杆释放窗口
release_stop_speed0.035 m/s判定已停稳的速度阈值

松杆释放（Release）机制

摇杆回中后触发 linear_release_active：参考立即置 0，但保留速度积分一段时间（最多 450 ms），直到实际速度低于 0.035 m/s。这样可避免松杆瞬间积分被清零导致的"点头"或反向冲量。

积分更新

∫ev += (vref − vactual) · dt，钳位 ±0.38

在 release 窗口内冻结线速度积分，防止参考突变为 0 时积分饱和。

6.2 偏航参考 update_yaw_reference

低通时间常数 τ=9 ms（比线速度更灵敏）。参考直接赋值：

ωref = −lpf(摇杆转向量)

取负号是因为手柄坐标系与机器人偏航正方向相反。积分钳位 ±0.55 rad·s 量级（累积误差积分）。

6.3 跳跃模式下的参考覆盖

跳跃时线速度/偏航参考不由摇杆决定，而由 jump_linear_vel_cmd 与 jump_turn_yaw_rate_cmd 注入（见第 9 节）。

7. 腿部与 Roll 控制

源码：controller.cpp::leg_loop()

7.1 控制目标

在平衡模式下，舵机腿承担两项任务：

  • 高度调节：通过 leg_height_base 改变站立高度（方向键 ↑↓，步进 0.025）。
  • Roll 补偿：PID 抑制机身左右倾斜（方向键 ←→ 调节 roll_adjust 偏置）。

7.2 舵机目标位置计算

roll_angle = LPF(MPU6050.roll_deg)
leg_position_add = PID(roll_angle − roll_adjust)   // Kp=8, Ki=30

left_pos  = 2048 + 8.4×(30 − leg_height_base) − leg_position_add
right_pos = 2048 − 8.4×(30 − leg_height_base) − leg_position_add

左右舵机镜像对称：左侧顺时针计数增大，右侧相反。结果钳位在 SERVO_LEFT_MIN/MAX 与 SERVO_RIGHT_MIN/MAX 范围内。

高度与 LQI 增益的耦合

腿高变化 → 更新 h → 重算 LQI 增益 K(h)。更高的机身（更小的 leg_height_base）对应更短的等效摆长，需要更大的反馈增益才能维持稳定。

8. 各模式详解

8.1 首次启动 FIRST_BOOT

PREPARE → 关闭舵机力矩 WAIT_FOR_SIGNAL → 等待 RB INIT → 舵机伸腿至站立位，reset()，begin_balance_recover() INIT_PREPARE → 350 ms 无电机/无平衡 INIT_RECOVER → balance_recover_loop() DONE → 进入 BALANCE

8.2 平衡模式 BALANCE

enable_balance = enable_steering = enable_motor = 1，每周期执行 leg_loop()。

按 LS（且无前进/后退摇杆输入）可复位：清除 roll_adjust、腿高基准、运动积分与 PID。

8.3 坐下模式 SIT

PREPARE → 舵机力矩模式 2（位置保持） MOVING → 左右电机 −0.2，机身前倾 退出条件：|pitch| ≥ 0.25 rad 或超时 5 s → 关闭电机 DONE → 可选 LS 关闭舵机力矩；RB 触发重新站起 EXIT_PREPARE / EXIT_RECOVER → 同启动恢复流程 EXIT → 回到 BALANCE

9.5 RECOVER 退出条件

同时满足姿态稳定（|pitch|<0.18 rad，|pitch_rate|<1.6 rad/s）与偏航到位（误差 <5° 或宽松判据），并持续 80 ms；或强制超时 350 ms / 220 ms 快速退出。

10. 平衡恢复（Balance Recover）

在启动站起、坐下后重新站起时调用，分两个阶段：

10.1 准备阶段（350 ms）

关闭平衡、转向与电机，清空运动参考，等待机械与传感器稳定。

10.2 恢复阶段

  • 开启平衡与电机，执行 leg_loop()，关闭转向。
  • LQI 只使用 pitch 两维（x₀、x₁），输出按时间 blend 渐入（220 ms）。
  • 退出条件：|pitch|<0.16 且 |pitch_rate|<1.2 持续 140 ms，且总时长≥250 ms；或最长 2500 ms 强制退出。
